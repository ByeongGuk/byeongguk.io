## 데이터의 이해

### 데이터의 정의
- 데이터는 개별 데이터 자체로는 의미 중요하지 않은 객관적인 사실(fact)
- 추론,예측,전망,추정을 위한 근거로 기능하는 특성
- 다른 객체와의 상호 관계 속에서 가치를 찾음

### 데이터 유형
- 정성적 데이터(Qualitative data) : 언어,문자(ex)주관식 응답)
- 정량적 데이터(Quantiative data) : 수치,기호,도형(ex)온도)

## 암묵지 VS 형식지

### 암묵지
- 체험을 통해 개인에게 습득
- 오랜 경험을 통해 개인에게 습득된 무형 지식

### 형식지
- 형상화 된 지식(예)회계,재무 관련 대차대조표에 요구되는 지식의 메뉴얼)

### 암묵지와 형식지 상호작용
- 지식경영 : 암묵지와 형식지가 나선화 형태로 회전하면서 생성,발전,전환되는 지식의 발전을 기반으로 한 기업의 경영
- 암묵지, 형식지의 4단계 지식전환 모드(**ㄱ(공통화),ㄴ(내면화) 사이에 표연(표출화,연결화)이 있다.**)
1. 공통화(암-암) : 암묵적 지식 노하우를 다른 사람에게 알려주는 것
2. 표출화(암-형) : 암묵적 지식 노하우를 책이나 교본 등 형식지로 만드는것
3. 연결화(형-형) : 책이나 교본에 알고 있는 새로운 지식(형식지)를 추가하는 것
4. 내면화(형-암) : 만들어진 책이나 교본을 보고 암묵적 지식을 습득

## 데이터와 정보읙 관계
### Data -> Infomation -> Knowledge -> Wisdom 계층 구조

Data : 가공하기전 Data

Information : 데이터의 가공 및 상관/연관 관계 속에서 의미가 도출

Knowledge : 정보 패턴을 이해하여 예측한 결과

Wisdom : 근본 원리에 대한 깊은 이해를 바탕으로 도출되는 아이디어

## 데이터베이스 특징
* 데이터베이스 : **통합,저장,공용,변화**되는 데이터
- 통합데이터 : 데이터가 중복x
- 저장데이터 : 저장매체에 저장
- 공용데이터 : 데이터를 공동이용
- 변화데이터 : 형상 변화하면서도 형상 현재의 정확한 데이터를 유지

## DBMS : 데이터베이스를 관리해주는 SW

### RDBMS
- 관계형 DB
- 정형화된 테이블로 구성된 데이터 항목들의 집합체
- MYSQL(오픈소스 RDBMS) , PL/SQL(상용 RDBMS)
- SQL : RDBMS의 데이터를 관리하기 위해 설계된 특수 목적의 프로그래밍 언어 챔벌린과 레이먼드 F.보이스가 개발

### ODBMS
- 객체지향 DB
- 객체들을 생성하여 계층에서 체계적으로 정리하고, 다시 계층들을 하위 계층이 상위계층으로부터 속성과 방법들을 물려받을수 있음.
- 복잡한 데이터 구조를 표현 및 관리하는 DBMS

### 참고사항
- Data Dictionary : 자료 저장소
- SQL : 데이터를 구축 및 활용하기 위한 언어
- ERD(ENTITTY RELATION DIAGRAM) : 실체와 이들의 관계를 도형으로 표현

## 데이터베이스 설계

### 데이터베이스 설계절차
- 요구조건분석 -> 개념적 설계 -> 논리적설계 -> 물리적 설계

- 요구조건분석 : 데이터베이스 사용자,사용목적,사용범위,제약조건등을 정리
- 개념적 설계 : E-R 모델,정보를 추상적 개념으로 표현
- 논리적 설계 : 자료를 컴퓨터가 이해할 수 있도록 특정 DBMS의 논리적 자료 구조로 변환
- 물리적 설계 : 논리적 설계를 물리적 구조 데이터로 변환

## NOSQL

### NoSQL : 관계적 데이터베이스 보다는 덜 제한적,디자인 단순화,수평적 확장성,세세한 통제 가능

### 저장방식 : MongoDB,Apache HBase,Redis

### 참고사항
- MongoDB : Binary JSON 문서형태로 저장하여, 여러 서버에 분산 저장 및 확장에 용이하며, 방대한 데이터 처리가 빠름. C++ 로 작성
- Apache HBase : 하둡 플랫폼을 위한 공개 비관계형 분산 데이터
- REDIS : Remote DICTIONARY SERVER : 키-값 값 구조의 데이터

## 기업 내부 데이터베이스 솔루션

### 시대별 기업 내부 데이터베이스 솔루션
- 1980 
    - OLTP(On-Line Transcation Processing) : 온라인 거래 처리 EX) 상품주문, 회원 정보 수정
    - OLAP(On-Line Analyayical Processing) : 온라인 분석 처리,다차원 데이터를 **대화식으로 분석**하기 위한 SW EX) 10년간 A사의 직급별 임금 상승률
- 2000 
    - CRM(Customer Relationship Management) : 고객의 이해를 돕고, 각종 마케팅 전략
    - SCM(Supply Chain Management) : 정보기술 활용,재고를 최적화 및 시간과 비용을 최적화 시키기 위함.

### 분야별 기업 내부 데이터베이스 솔루션 - 제조부문

- Data Warehouse : 하나의 통합된 데이터 저장 공간,시간적 흐름에 따라 변화하는 값
    * 데이터의 통합 : 전사적 차원
    * 데이터의 시계열성 : 시간의 흐름에 따라 변화하는 값
    * 데이터 주체 지향적,비소멸성 : 주제에 따라 분류,저장,관리

- Data Mart : **소규모 단일 주제**의 데이터 웨어 하우스,특정 조직의 특정 업무 분야에 초점
- ERP(Enterprise Resource Planning) : 주요 프로세스 관리를 돕는 여러 모듈로 구성된 통합 APP
- BI(Business Intelligence)
    * **데이터를 통합/분석하여 기업 활동에 연관된 의사결정**을 돕는 프로세스
    * **가트너**는 사용자가 필요로 하는 정보를 정확한 시간에 제공할 수 있는 환경
    * 하나의 특정 비지니스 질문에 답변
- BA(Business Analytics): **통계적/수학적인 분석에 초점**,BI 보다 진보된 형태
- ad hoc report : **BI와 빅데이터 분석의 차이점을 표현한 키워드**,일회용으로 작성된 임시보고서(**빅데이터 분석 키워드 :Optimzation,forecast,insight**)

### 분야별 기업 내부 데이터베이스 솔루션 - 금융부문

- 블록체인 : 중앙 집중형 서버에 거래 기록을 보관하는 방식에서 벗어나 거래에 참여하는 모든 사용자에게 거래 내용을 보내주며 데이터 위조 방지 기술(그 외에는 EIA,EDW,ERP,e-CRM이 있음.)

### 분야별 기업 내부 데이터베이스 솔루션 - 유튱부문

- KMS(Knowledge Management System) : 조직내의 지식을 체계적으로 관리 시스템
- RFID : 무선주파수를 활용한 대상 식별 기술

## 빅데이터

### 빅데이터란?  :DB SW 저장,관리,분석 할 수 있는 범위를 초과하는 규모의 데이터
- 4V(Volume,Variety,Velocity,Value)
    - VOLUME : 데이터의 크기
    - VARIETY : 데이터의 다양성,정형화된 데이터를 넘어 모든 유형의 데이터
    - Velocity : 데이터의 속도
    - Value : **'Value'는 비지니스 효과 요소 , Volume,Variety,Velocity는 '투자 요소'**

### 빅데이터 역할
- 석탄/철 : 서비스 분야의 생산성
- 원유 : 산업 전반의 생산성
- 렌즈 : Data의 디지털화
- 플랫폼 : 공동 활용의 목적으로 구축된 유/무형의 구조물

### 빅데이터 가치 선정, 본질적 변화

- 빅데이터 가치 산정이 어려움
    - 데이터 활용 방식
    - 새로운 가치 창출
    - 분석기술의 발달
- 빅데이터 만들어내는 본질적인 변화
    - 사전처리(표준화된 문서) -> 사후처리(DATA GET 후 가치 확인)
    - 표본조사 -> 전수조사
    - 질 -> 양
    - 인과관계 -> 상관관계

### 빅데이터 활용 기법
- 연관 규칙 학습 : 변수간 주목할만한 상관관계가 있는가(EX) 우유구매자가 기저귀 판매 상관관계)
- 유형 분석 : 사용자는 어떤 특성을 가진 **집단**에 속하는가
- 유전 알고리즘 : **최적화가 필요한 문제의 해결책**을 자연선택,돌연변이 등과 같은 메커니즈을 통해 점진적으로 진화(EX) 어떤시간대에 방송배치)
- 기계학습 : 훈련 데이터로부터 패턴 학습을 통해 예측
- 회귀분석 : 선형함수로 나타낼수 있는 수치데이터 분석
- 감정분석 : 감정 분석(EX)후기)
- 소셜네트워크 분석 : 영향력 있는 사람을 통해 소셜 관계 파악

### 빅데이터 위기요인과 통제방안
- 사생활 침해 : 동의제 -> **책임제**
- 책임 원칙 훼손 : **책임 강화 원칙**
- 데이터 오용 : 데이터 **알고리즘**에 대한 접근권 허용

### 개인정보 비식별화 기법
- 데이터 마스킹
- 데이터 범주화 : 홍길동,35세 -> 홀씨,30-40세
- 가명 
- 잡음 첨가 
- 총계처리/평균값 대체
- 데이터 값 삭제

### 데이터 사이언스
- 데이터 사이언스의 정의 : 다양한 유형의 데이터,분석뿐만 아니라 이를 효과적으로 구현하고 전달하는 과정까지 포함한 포괄적 개념,총체적 접근법 사용
- 데이터 핵심 수구성요소 : IT,분석,비지니스 컨설팅

|      |데이터 사이언스|통계학|데이터 마이닝|
|------|---|------|----------|
|분석 대상|정형,비정형 등 다양한 데이터 유형|정형화된 데이터|---|
|분석 방법|분석+시각화+전달 포함한 포괄적 개념|---|분석에 초점|
|학문 접근|종합적 학문 또는 총체적 접근법|---|---|

#### 데이터 사이언스 역량 : 하드스킬 & 소프트 스킬
- 가트너가 본 데이터 사이언스 역량은 데이터 관리,분석,모델링,비지니스 분석,소프트 스킬
- 하드스킬 : Machine Learning,Modeling Data,Data Technical Skill
- 소프트스킬 :창의적 사고,호기심,논리적 비판(Storytelling,Visualizing)
