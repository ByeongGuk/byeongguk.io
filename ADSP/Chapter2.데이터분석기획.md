# 데이터 분석 기획
## 데이터 분석 기획의 이해
### 분석 기획(**분석대상을 알면 OS,방법을 알면 ID**)
- 분석 대상(what), 분석의 방법(how)에 따라 4가지 구분
    - Optimization(최적화) : 대상o, 방법o 
    - Solution(솔루션) : 대상o, 방법x
    - Insight(통찰) : 대상x, 방법o
    - Discovery(발견) : 대상x, 방법x

### 목표 시점별 분석기획 방안
- 단기 : Quick & Win, Speed & test, Problem Solving
- 장기 : Accuracy & Deploy, Long Term View, Problem Defintion

### 분석 기획 시 고려사항
- 가용한 데이터 : 데이터를 유형으로 분류 정형,비정형,반정형
- 적절한 유스케이스
- 장애 요소들에 대한 사전 계획 수립

### 데이터 유형, 저장 방식
- 데이터 유형 분류
    - 정형 데이터 : ERP,CRM Transaction Data,Demand Forecast
    - 반정형 데이터 : Competitor Pricing, Sensor, Machine Data
    - 비정형 데이터 : Email,SNS,Voice,IOT,보고서,뉴스
- 데이터 저장방식
    - RDB: 관계형(ORACLE,MSSQL,Mysql)
    - NOSQL :비관계형(Mongo DB, Cassandra,Hbase, Redis)
    - 분산파일시스템 : 분산된 서버의 디스크에 파일 저장(HDFS)

### 분석 방법론 
- 구성요소 : 상세한 절차, 방법, 도구와 기법, 템플릿과 산출물
* 참고사항 
    - 프레이밍 효과 : 동일한 사항이라도 방법에 따라 해석 또는 의사결정이 달라지는 왜곡 현상
- 분석 방법론 모델 3가지
    - 폭포수 모델 :**하향식 진행**,피드백 수행
    - 나선형 모델 : 반복을 통해 검증적 개발
    - 프로토타입 모델 : 정확히 규정하기 어렵고,데이터 소스도 명확히 파악하기 힘들떄 사용,**상향식 접근방법**,

- KDD 분석 방법론
    - DB에서 의미 있는 지식을 탐색하는 데이터 마이닝 프로세스
    - 단계 : 데이터셋 -> 데이터 전처리(Noise,Outlier,Missing Value) -> 데이터 변환(변수 선택,데이터의 차원 축소) -> 데이터 마이닝(데이터 패턴) -> 결과 평가
- CRISP-DM 분석 방법론(Cross-Industry Standard Process for Data Mining)
    - 단계 : 업무 이해 -> 데이터 이해 -> 데이터 준비 -> 모델링 -> 평가 -> 전개
    - 업무 이해 : 업무 목적 파악 -> 상활파악 -> 데이터마이닝 목표설정 -> 프로젝트 계획 수립
    - 데이터 이해 : 데이터 수집,데이터 탐색, 데이터 품질확인
    - 데이터 준비 : 데이터 전처리
    - 모델링 : 다양한 모델링 기법과 알고리즘 선택, 파라미터 최적화, 모델링 기법 선택, 모델링 작성, 모델 평가
    - 평가 : 모델링 과정 평가,적용성 평가
    - 전개 : 실제 업무에 적용

### 빅데이터 분석 방법론
- 순서 : 분석기획 -> 데이터 준비 -> 데이터 분석 -> 시스템 구현 -> 평가 및 전개
    - 분석 기획 : 비지니스 이해 및 범위 설정(SOW), 프로젝트 정의 및 계획 수립(WBS), 프로젝트 위험 계획 수립(대응방법:회피/전이/완화/수용)
    - 데이터 준비 : 필요 데이터 정의(정형/비정형/반정형)(내부 데이터/외부 데이터), 데이터 스토어 설계(관계형/비관계형DB), 데이터 수집 및 적합성 점검(데이터 수집 및 저장,데이터 정합성 점검)
    - 데이터 분석 : 분석용 데이터 준비(데이터 준비/분석 반복적 수행), 텍스트 분석, 탐색적 분석, 모델링, 모델 평가 및 검증
    - 시스템 구현 : 설계 및 구현, 시스템 테스트 및 운용
    - 평가 및 전개 : 모델 발전 계획, 프로젝트 평가 보고, 평가 및 전개

### 분석 과제 도출 방법
- 하향식 접근방법 : 문제가 확실할때 사용
    - 문제 탐색(Problem Discovery): 비지니스 모델 기반(업무,제품,고객,지원 인프라,규제와 감사),외부 참조 모델 기반 문제 탐색(벤치마킹),분석 유즈 케이스(기대 효과)
    - 문제 정의(Problem Definition) : 데이터 분석 문제 변환(데이터 및 기법을 정의하기 위한 데이터 분석 문제로 변환)
    - 해결 방안 탐색(Solution Search) : 수용 옵션 도출
    - 타당성 검토(Feasibility Study) : 타당성 평가,과제 선정
- 상향식 접근방법 : 문제의 정의가 어려울때 사용
    - 문제의 정의 자체가 어려운 경우
    - 비지도학습 방법에 의해 수행
- 디자인 싱킹 : 상/하향식 반복적 사용

### 지도학습 VS 비지도 학습
- 지도학습(Surpervised Learning) : 명확한 Input/Output, 예측, 분류
- 비지도학습(Unsupervised Learning) : 컴퓨터 분류,군집화 

### 분석 프로젝트의 특징
- 분석 프로젝트의 특징
    - 영역별 관리
    - 5가지 주요특성(Data Size,Data Complexity, Speed, Analyic Complexity, Auucracy & Precsion)
        - Data Size : 데이터의 양을 고려하는 관리하는 수립 필요
        - Data Complexity : 분석 모델 선정에 대한 고려 필요
        - Speed : 일,주 단위 실적은 배치 형태 작업/ 사기 탐직,서비스 추천은 실시간
        - Analytic Complexity : 정확도와 복잡도는 트레이드 오프
        - Accuray & Precision : Accuracy(모델과 실제값의 비교), Precison(안정성)

### 10개 주제별 프로젝트 관리 체계
- 시간, 범위, 품질, 통합, 이해관계자, 자원, 원가, 리스크, 조달, 의사소통

### 분석 마스터 플랜
- 분석 마스터 플랜 수립 프레임워크
    분석과제 -> 우선순위 고려요소(전략적 중요도,ROI(투자자본수익률),실행용이성) -> 적용 우선순위 결정(업무 내재화,분석데이터 적용기준,기술적용수준)-> 분석 구현 로드맵 수립

#### ROI 관점에서의 빅데이터 4V
- Volume : 크기
- Variety : 종류
- Velocity : 속도
- Value : 분석 결과 활용 및 실행을 통한 비지니스 가치

### 분석 과제 우선순위 선정 기법
- 포트 폴리오 사분면을 통한 과제 우선순위 선정 : 난이도(분석 비용,적용 범위 측면)/시급성(전략적 중요도 및 목표가치)에 따라 구분(3사분면 먼저 후 시급성/난이도에 따라서 진행)

### 분석 거버넌스(기업, 비영리 기관 드엥서 규칙,규범 및 행동이 구조화,유지,규제) 체계 수립
- 거버넌스 체계 개요 : Process,Organzation,System,Human Resource,Data (**비용 및 예산은 제외**)
- 데이터 분석 준비도 : 데이터 준비 준비도 프레임 워크 : 분석 업무파악(예측 분석,시뮬레이션,최적화 분석),입력 및 조직, 분석 기법(분석 기법 라이브러리),분석 데이터, 분석 문화, 분석 인프라(EAI,ETL)
- 분석 성숙도 모델 :도입 단계,활용 단계,확산 단계,최적화 단계
- 분석 수준 진단결과 : 성숙도/준비도 에따라 4사분면 분석(성숙도UP/준비도UP: 확산형,성숙도UP/준비도Dn:정착형,성숙도Dn/준비도UP:도입형,성숙도Dn/준비도Dn:준비형) 
- 분석 지원 인프라 방안 수립 : 하드웨어/운영체제/분석엔진,분석라이브러리/데이터 체리 프레임 워크/분석 서비스 제공 API/분석 APP/분석 서비스 제공 엔진
- 데이터 거버넌스 체계 수립 : 데이터 표준화(데이터 표준용어,명명규칙,메타 데이터,데이터 사전 구축),데이터 관리체계(관리 원칙 수립), 데이터 저장소 관리(사전영향평가), 표준화 활동(모니터링)
- 데이터 조직 및 인력 방안 수립
    - 집중형 조직 구조 : 독립적인 분석,모든 분석 업무 담당 But, 기능중식/분산 조직 구조와 중첩하지 않도록 유의
    - 기능중심 조직 구조 : 해당 업무부서에서 직접 분석
    - 분산 조직 구조 : 신속한 업무에 적합    
- 분석 과제 관리 프로세스 수립 :과제 발굴,과제 수행
- 분석 교육 및 변화 관리

### 빅데이터 거버넌스
- 빅데이터 거버넌스 특징 : 분석 대상 및 목적을 명확히 정의,데이터 주기/관리방안, ERD는 계속해서 변경사항 관리, 산업/데이터/정보 유형별 구분

### 관련 용어
- Servitization : 제조업과 서비스업의 융합
- CoE(Center of Excellence) : 조직 내 분석 전문조직
- ISP(정보전략계획) : 전사적인 종합추진계획
- Sandbox : 외부 접근 차단
